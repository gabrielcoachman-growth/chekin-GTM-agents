# Language Config — Spanish (ES)

## Register
- **Formality:** Informal `tú` throughout — this is the established Chekin voice for Spanish
- Apply `tú` conjugations consistently: "automatiza", "gestiona", "prueba", "empieza" — never "automatice", "gestione", "pruebe"
- Exception: if the page targets enterprise hotel chains, switch to `usted` — this will be indicated in the filename or folder name (`/hotel-enterprise/`)
- Never mix registers on the same page

## Chekin voice — Spanish adaptation
- **Problem-first framing:** Lead with the guest or host's pain, not the feature. "Los check-ins manuales te quitan tiempo" before "Chekin automatiza el proceso"
- **Outcome-led language:** Focus on what the property manager achieves. "Tus huéspedes hacen el check-in solos" not "Chekin ofrece check-in online"
- **North star phrase equivalent:** "funcione solo" → use "funcione solo" directly in ES — it is already Spanish and natural
- **Brevity:** Spanish marketing copy tends toward longer sentences than English. Cut them. Shorter sentences convert better.
- **Brand name:** Never add article before Chekin — not "el Chekin", not "la plataforma Chekin" in headlines. Just "Chekin"

## Common translation pitfalls to fix
These are patterns that Claude Code tends to produce when translating from English. Fix all of them:

- **"Bienvenido a Chekin"** as a page headline → rewrite to outcome-led. "Automatiza el check-in de tus huéspedes" or similar
- **"Nuestra plataforma ofrece..."** → rewrite to "Con Chekin, tus huéspedes..."
- **"Haga clic aquí"** or **"Haga clic"** → never. Use "Empieza gratis", "Pruébalo ahora", "Ver cómo funciona"
- **"Por favor, complete el formulario"** → "Completa el formulario" (tú + no "por favor" in CTAs)
- **"Solución"** as a standalone noun in headings → vague. Replace with what the solution actually does
- **"Gestión de propiedades"** as an H1 → too generic for SEO and not outcome-led
- **Gerund-heavy copy:** "Estamos ofreciendo...", "Estamos ayudando..." → rewrite to active: "Ofrece...", "Ayuda..."
- **False cognate "actual"** → "actual" in Spanish means "current/present", not "actual/real". Check every instance
- **"Requerimientos"** → use "requisitos"
- **"Aplicación"** when referring to the Chekin product → use "herramienta" or "plataforma" or just "Chekin". "App" is fine for mobile context

## SEO — Spanish market patterns
- Primary search intent for Chekin in ES: informational + transactional ("check-in online alquiler vacacional", "automatizar check-in turístico")
- Include "alquiler vacacional" or "alquiler a corto plazo" in meta description where relevant
- "Checkin" (no hyphen) is a common search variant — acceptable in meta description but keep "check-in" in visible body copy
- Avoid keyword cannibalization with "gestión de alojamientos" — too broad; be specific about the function
- Spanish users search in Spanish — do not leave English keyword phrases in meta tags even if they are industry terms
- Local market note: Spanish STR market has specific regulations (tourist license, Ley de Arrendamientos Urbanos). If the page references compliance, use "normativa" not "regulación"

## Quality check — native speaker test
Before finishing, read the H1, subheadline, and first paragraph aloud. Ask: would a Spanish property manager from Madrid or Barcelona say this? If it sounds translated, it needs another pass.
