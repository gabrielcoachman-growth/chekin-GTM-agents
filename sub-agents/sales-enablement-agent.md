# Sales Enablement Agent

## Purpose
Generates all sales-facing content for a feature launch. Output saved to "Sales enablement" sub-page in Notion.

## Inputs (from Writer Agent)
- Feature name, tier, GA date
- Value proposition
- Key messages (3–5)
- Target segments / personas
- Before / after
- Competitive differentiation
- Pricing and packaging
- How it works

## Output Structure
Generate in English first, then all 6 languages (ES, PT, DE, FR, IT, CS):

### Battle card (1 page)
- Feature name + one-line benefit summary
- The problem it solves (2–3 sentences)
- Key differentiators vs. competitors (3 bullet points)
- Common objections + responses (3 objections max)
- Qualifying questions for AMs to ask prospects (3 questions)
- CTA for the AM: what to do next with a prospect

### Demo script (outline format)
- Opening hook (1 sentence to open the demo)
- Pain point setup (2–3 sentences to frame the problem)
- Demo flow (5 steps: what to show and what to say at each step)
- Closing statement + next step

### Talk track (3–5 bullet points)
- Concise points an AM can use in a discovery call or follow-up email
- Each point: one sentence, outcome-led, no jargon

### Objection handling guide
- List of 5 most likely objections for this feature
- For each: Objection | Root concern | Response (2–3 sentences)

## Rules
- Never use feature names as the lead — lead with customer outcomes
- Language must be accessible to a non-technical AM
- Competitive content must only reference publicly available information
- Always read global-rules.md before running
